Contributing to CareSignal

Thank you for your interest in contributing to CareSignal. This project is a minimal starter intended for prototyping patient-facing dashboards. Please follow these guidelines:

- Code style: run `npm run format` if available, otherwise follow the existing code style.
- Branching: create feature branches named `feat/<short-name>` or `fix/<short-name>`.
- Commits: write concise commit messages and reference issues when applicable.
- Accessibility: ensure UI components meet basic accessibility (semantic elements, ARIA where needed).
- Data privacy: do not commit any real PHI (Protected Health Information) into the repository.

Submitting PRs

- Include a short description of the change and any migration steps.
- Add screenshots or animated GIFs for UI changes when possible.
- Tag review requests for clinical or security reviewers if the change touches data handling or authentication.
